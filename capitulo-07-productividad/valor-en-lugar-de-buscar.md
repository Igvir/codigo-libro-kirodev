## Documentación Interna (vía MCP Confluence)

Tenemos MCP de Confluence configurado.

Antes de implementar features nuevos:
- Busca en Confluence si ya existe doc de arquitectura
- Espacios relevantes: "Engineering", "Product Specs", "API Docs"

Ejemplo de búsqueda:
"Busca en Confluence documentación sobre sistema de notificaciones"

Kiro encontrará docs relevantes y los aplicará al código generado.
